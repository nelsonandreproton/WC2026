⚽ WC2026 Prediction Bot

Python 3.12 python-telegram-bot 22 SQLAlchemy 2.x · SQLite APScheduler Docker

Bot de Telegram para um bólão do Campeonato do Mundo de Futebol 2026. Cada jogador prevê o resultado exato de cada jogo, recebe por DM o resultado e os seus pontos, e vê a classificação ao fim de cada ronda. Ninguém vê as previsões dos outros.

Como pontua

AcertoPontos
Direção (vitória / empate / derrota)+3
Resultado exato+2 extra (total 5)
Campeão do Mundial (aplicado após a final)+100
  • Empate conta como direção própria.
  • Mata-mata pontua pelo resultado aos 90' (prolongamento e penáltis ignorados).
  • O campeão é resolvido por quem realmente avançou (inclui penáltis).
  • Previsões fecham LOCK_MINUTES antes do apito (default 15).
  • A aposta no campeão fecha no apito do jogo de abertura.

Arquitetura

scoring.py
Pontuação pura (direção / exato / campeão). 100% testado.
service.py
Lógica de negócio sobre a sessão (registo, prever, campeão).
rounds.py
Identidade canónica de ronda a partir de (stage, matchday).
standings.py
Classificação, conclusão de ronda, bónus de campeão.
notifier.py
Envio de DMs com throttling (respeita limites do Telegram).
providers/
Abstração FootballProvider + football-data.org + sync idempotente.
bot/
Handlers Telegram (finos) + formatação (escape de Markdown).
scheduler/
Jobs: sync, poll+score, publicar tabelas, campeão.

Fonte de dados

Resultados e calendário via football-data.org (tier grátis), atrás de uma interface FootballProvider — trocar para outra API (ex.: API-Football) é uma única classe nova. Polling em janelas de jogo, com backoff e sem retry em HTTP 429. Os mata-mata vêm com matchday=null, por isso a ronda é identificada pelo stage, não pelo matchday.
